# Phase 3 – Step 1: Creating the HTTP Response Module

## Objective

In this step, we introduce a **Response abstraction** for our Rust HTTP server.
Previously, responses were manually constructed using formatted strings. This approach becomes difficult to maintain as the server grows.

The goal of this step is to create a **structured Response object** that will represent every HTTP response generated by the server.

This design is similar to how modern Rust web frameworks like Actix Web and Axum internally structure responses.

---

# Why This Step Is Important

Before this step, responses were created manually:

```
HTTP/1.1 200 OK
Content-Type: text/plain
Content-Length: 17

Hello from Rust
```

Constructing responses manually leads to:

* Hard-to-maintain code
* Repeated formatting logic
* Limited extensibility

By introducing a **Response struct**, we separate the **response representation** from the **serialization logic**.

# Step 3: Define the Response Struct

Inside `response.rs`:

```rust
use std::collections::HashMap;

pub struct Response {
    pub status_code: u16,
    pub headers: HashMap<String, String>,
    pub body: String,
}
```

## Field Explanation

| Field       | Purpose                                |
| ----------- | -------------------------------------- |
| status_code | HTTP status code (200, 404, etc.)      |
| headers     | HTTP headers stored as key-value pairs |
| body        | Response body content                  |

---

# Step 4: Add Constructor

We create a constructor that initializes the response object.

```rust
impl Response {

    pub fn new(status_code: u16) -> Self {

        Response {
            status_code,
            headers: HashMap::new(),
            body: String::new(),
        }
    }

}
```

---

# Step 5: Add Header Builder Method

This method allows adding headers to the response.

```rust
pub fn header(mut self, key: &str, value: &str) -> Self {

    self.headers.insert(key.to_string(), value.to_string());

    self
}
```

This enables **method chaining**.

Example:

```rust
Response::new(200)
    .header("Content-Type","text/plain");
```

---

# Step 6: Add Body Builder Method

The body method sets the response content.

```rust
pub fn body(mut self, body: &str) -> Self {

    self.body = body.to_string();

    self
}
```

Example:

```rust
Response::new(200)
    .header("Content-Type","text/plain")
    .body("Hello from Rust");
```

# Final response.rs Implementation

```rust
use std::collections::HashMap;

pub struct Response {
    pub status_code: u16,
    pub headers: HashMap<String, String>,
    pub body: String,
}

impl Response {

    pub fn new(status_code: u16) -> Self {

        Response {
            status_code,
            headers: HashMap::new(),
            body: String::new(),
        }
    }

    pub fn header(mut self, key: &str, value: &str) -> Self {

        self.headers.insert(key.to_string(), value.to_string());

        self
    }

    pub fn body(mut self, body: &str) -> Self {

        self.body = body.to_string();

        self
    }

}
```

# What This Enables

The Response struct allows us to write cleaner response code:

```
Response::new(200)
    .header("Content-Type","text/plain")
    .body("Hello from Rust")
```

Instead of manually formatting HTTP responses.
